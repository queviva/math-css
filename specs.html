various tags and options for illuminated markup
  • c-omm an expandable comment allows for short comments to expand into larger, scrollable descriptions on touch or mouseover 

    <c-omm>
    short comment
    <b>
        anything inside b tag will expand
    </b>
</c-omm>
  • f-rak a simple fraktion tag
    x2 3 5 + 9
    allows for extremely simple fraktions, or more complicated if needed 
    <f-rak
    data-viva   // illuminates only the bar
    data-slash1 // optional slash-type bar
>
    <b> numerator </b>
    <b> denominator </b> <!-- b optional -->
</f-rak>

    if the denominator has no other tags, its b tag is optional - for instance, this would display 3/5:
    <f-rak><b>3</b>5</f-rak>
  • s-umm sigma notation for summation
    3 5x2 0 ∞ + 9 

    <s-umm
    data-viva  // illuminates everything
    data-sigma // illuminates only sigma
    data-from  // illuminates only from value
    data-to    // illumiates only to value
    data-ghost // displays in outline
>
    <b> thing to sum </b>
    <b> from </b> <!-- optional -->
    <b> to </b> <!-- optional -->
</s-umm>
  • rite-dist distribute-right gestures
    3 x23 + 53
    indicates that the leading coefficient has moved into the other terms, and displays it being multiplied - its markup is more involved, though all data parameters and styling are optional 
    <rite-dist
    data-dark  // no illumination
    data-norm  // regular font
    style="
        --arc1-wide: // first arc width
        --arc2-wide: // second arc width
    "
>
    <b> leading coefficient </b>
    <b> first term
        <b> ...muliplied by</b> <!-- optional -->
    </b>
    <b> second term
        <b> ...muliplied by</b> <!-- optional -->
    </b>
    
</rite-dist>
  • left-fakt leftward faktorization gestures
    3 (3x + 9)
    pulls a faktor out of two terms and moves it to the left - arc distances and be adjusted with css vars in the style 
    <left-fakt
    data-dark  // no illumination
    data-vapor // vaporizes the gesture
    data-norm  // uses normal font
    style = "
        --arc1-wide: 46%  // first arc width
        --arc2-wide: 100% // second arc width
    "
>
    <b> faktor </b>
    expression
</fakt-left>
  • p-owr derivative power rule gestures
    3x5
    presents illuminated gestures for using the power rule to compute a derivative - it recognizes a <sup> tag 
    <p-owr
    data-dark    // not illuminated
    data-ghost   // ghosts out the gestures
    data-opt#    // uses different gestures
    data-margin# // increases right margin
>
    3x<sup>2</sup> <!-- use sup for power -->
</p-owr>
  • i-pow power rule for integration
    3x5 dx
    creates gestures for computing the integral using the power rule 
    <i-pow
    data-dark // no illumination
>
    3x<sup>5</sup>
</i-pow>
  • l-ogb logarithm with base
    35x
    logarithm with optional base and gestures to indicate how to solve for variable encased in a log cabin 
    <l-ogb
    data-viva      // illuminates entire log
    data-viva-base // illuminates only base
    data-ghost     // ghosts out
    data-arc       // shows solving gesture
    data-hat       // shows solving hat
    data-gank      // scribbles it out
>
    <b> base </b> <-- optional -->
    thing to log
</l-ogb>
  • v-iva illuminated text
    illuminating
    anything wrapped in a viva tag will recieve the illumination filter 
    <v-iva> anything </v-iva>
  • [data-viva] parameter for luminance
    vibrance
    many tags illuminate through the use of the data-viva parameter, and may have several other illuminating or gesturing options 
    <b data-viva>
    vibrant text
</b>
  • r-ant longer comments a longer, indepth comment that does not expand but remains fully displayed 
    <r-ant>
    longer, non-expanding comment
</r-ant>
  • up-dn sub- & super- script
    3x 2
    i     + 5
    allows for powers and subscripts simultaneously on the same item, formatted correctly on the line - it avoids the errors cause by using both <sup> and <sub> on the same item 
    <up-dn>
    exponent
    <br>
    subscript
</up-dn>
  • p-ren parenthetical enclosure, ()'s
    3 x2 + 5
    treats the parenthesis around, and the items within an enclosure, as an object that can have gestures and illuminations applied to the various parts

    this functions the same as the <b data-pren> option 
    <p-ren
    data-viva      // illuminates ()'s
    data-viva-all  // illuminates everthing
    data-ghost     // ghosts the ()'s
    data-ghost-all // ghosts entire enclosure
    data-vapor     // vaporizes the ()'s
    data-vapor-all // vaporizes everything
>
    thing to enclose
</p-ren>
  • b-rak brakette enclosure, []'s
    3 + x(x + 5) + 7y
    larger, taller, brakette enclosure for encompassing statements which are themselves often taller than simple []'s and accomodates any exponents that may follow it 
    <b-rak
    data-viva      // illuminates []'s
    data-viva-all  // illuminates everything
    data-ghost     // ghosts []'s
    data-ghost-all // ghosts everything
    data-vapor     // vaporizes []'s
    data-vapor-all // vaporizes everything
    data-tall      // makes []'s extra tall
    data-norm      // makes []'s normal height
>
    thing to enclose
</b-rak>
  • c-url curved group indicator
    3 + 5x + 7x = 9y + y 
    <c-url
    data-dark  // removes illumination
    data-ghost // ghost out the curl
    data-vapor // vaporizes the curl
    data-lo    // places the curl underneath
    data-opt1  // alternative squared shape
    
>
    grouped items
</c-url>
  • s-umm sigma notation
    3 θ2 0 2π
    sigma notation for summation 
    <s-umm
    data-viva       // illuminates everything
    data-viva-sigma // illuminates the sigma
    data-viva-from  // illuminates from val
    data-viva-to    // illuminates to val
    data-ghost      // ghosts the notation
    data-scrib      // scribbles out
>
    thing to sum
    from <!-- optional -->
    to <!-- optional -->
</s-umm>
  • e-kwa something "equals" another
    x2 + ( 1x + 2x 3x ) + 2
    places some term directly overtop of another, typically indicating that the one expression will replace the other, because of their ekwivalence 
    <e-kwa
    data-dark // removes default illumination
    data-norm // normal font
>
    <b> old val </b>
    <b> replacement </b>
</e-kwa>
  • r-dux cancel and reduce|replace
    sec2 (1 + tan2)
    cancels out an item and displays an optional replacement value 
    <r-dux
    data-dark    // non-illuminated
    data-norm    // default font
    data-viva-to // illuminates only redux val
    data-opt#    // alternative shaped slash
    data-cirk    // circle-shaped slash
    data-hi      // slashes higher
    data-top     // redux val on top
>
    <b> reduce-to val </b> <!-- optional -->
    item to cancel
</r-dux>
  • c-ank cancel; short-hand for r-dux
    (3x2 + 5)
    simply another version of <r‑dux> that cancels out an item but does not replace it with anything; identical to using <r‑dux> without the optional redux value 
    <c-ank
    data-dark  // non-illuminated slash
    data-hi    // higher slash
    data-opt#  // alternative shaped slash
    data-scrib // scribbled out
>
    item to cancel
</c-ank>
  • cirk encircles an item
    3 (-1 + sec2) =tan2
    puts an illuminated encircling gesture around an item; options allow for several types of gestures, or a note 
    <c-irk
    data-dark // no illumination
    data-vers // inverse illumination
    data-opt# // alternative shapes
    data-flip // mirrors the shape
    data-turn // rotates the shape
    data-norm // default font on note
>
    things to encircle
    <n-ote> note </n-ote> <!-- optional -->
</c-irk>
  • e-val evaluate expression in []'s
    3 x22 0 3

    brakette notation for evaluation 
    <e-val
    data-viva      // illuminates everything
    data-viva-from // illuminates from val
    data-viva-to   // illuminates to val
>
    expression
    <b>from</b> <-- optional -->
    <b>to</b> <-- optional -->
    
</e-val>
  • i-fak illuminated faktorization
    9x3 3, 3, x, x, x + 12 1 12,
    2 6,
    3 4     - 3x2 3, x,  
    ghosts out a value and presents illuminated versions of its faktors ontop of it, allowing you to see both the value and its faktors together at once 
    <i-fak
    data-dark  // faktors not illuminated
    data-vapor // value vaporized
    data-cirk  // adjusts circled faktors
>
</i-fak>
