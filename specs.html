various tags and options for illuminated markup
  • c-omm an expandable comment allows for short comments to expand into larger, scrollable descriptions on touch or mouseover 

    <c-omm>
    short comment
    <b>
        anything inside b tag will expand
    </b>
</c-omm>
  • f-rak a simple fraktion tag
    x2 3 5 + 9
    allows for extremely simple fraktions, or more complicated if needed 
    <f-rak
    data-viva   // illuminates only the bar
    data-slash1 // optional slash-type bar
>
    <b> numerator </b>
    <b> denominator </b> <!-- b optional -->
</f-rak>

    if the denominator has no other tags, its b tag is optional - for instance, this would display 3/5:
    <f-rak><b>3</b>5</f-rak>
  • s-umm sigma notation for summation
    3 5x2 0 ∞ + 9 

    <s-umm
    data-viva  // illuminates everything
    data-sigma // illuminates only sigma
    data-from  // illuminates only from value
    data-to    // illumiates only to value
    data-ghost // displays in outline
>
    <b> thing to sum </b>
    <b> from </b> <!-- optional -->
    <b> to </b> <!-- optional -->
</s-umm>
  • p-ren parenthetical enclosure
    3 x2 + 1
    parenthesis that allows for manipulation of the enclosed items or just the parenthesis themselves 
    <p-ren
    data-viva      // illuminates parenthesis
    data-viva-all  // illuminates everything
    data-ghost     // ghosts out the parenthesis
    data-ghost-all // ghosts out everything
    data-vapor     // vaporizes parenthesis
    data-vapor-all // vaproizes everything
>
    enclosed terms
</p-ren>
  • fakt-left leftward faktorization gestures
    3 (3x + 9)
    pulls a faktor out of two terms and moves it to the left 
    <fakt-left
    data-dark  // no illumination
    data-ghost // ghosts out the gesture
    data-vapor // vaporizes the gesture
>
    <b> faktor </b>
    expression
</fakt-left>
  • dist-rite simple distribution to the right
    3 (x + 5) 
    <dist-rite> expression </dist-rite>
  • rite-dist elaborate distribute-right gesture
    3 x23 + 53
    indicates that the leading coefficient has moved into the other terms, and display it being multiplied - its markup is more involved, though all data parameters and styling are optional 
    <rite-dist
    data-dark  // no illumination
    data-norm  // regular font
    style="
        --arc1-wide: // first arc width
        --arc2-wide: // second arc width
    "
>
    <b> leading coefficient </b>
    <b> first term
        <b> ...muliplied by</b>
    </b>
    <b> second term
        <b> ...muliplied by</b>
    </b>
    
</rite-dist>
  • p-owr derivative power rule gestures
    3x5
    presents illuminated gestures for using the power rule to compute a derivative - it recognizes a <sup> tag 
    <p-owr
    data-dark    // not illuminated
    data-ghost   // ghosts out the gestures
    data-opt#    // uses different gestures
    data-margin# // increases right margin
>
    3x<sup>2</sup> <!-- use sup for power -->
</p-owr>
  • i-pow power rule for integration
    3x5 dx
    creates gestures for computing the integral using the power rule 
    <i-pow
    data-dark // no illumination
>
    3x<sup>5</sup>
</i-pow>
  • l-ogb logarithm with base
    35x
    logarithm with optional base and gestures to indicate how to solve for variable encased in a log cabin 
    <l-ogb
    data-viva      // illuminates entire log
    data-viva-base // illuminates only base
    data-ghost     // ghosts out
    data-arc       // shows solving gesture
    data-hat       // shows solving hat
    data-gank      // scribbles it out
>
    <b> base </b> <-- optional -->
    thing to log
</l-ogb>
